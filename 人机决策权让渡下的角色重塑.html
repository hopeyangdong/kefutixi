一项目背景及我的角色

处理场景：迟滞场景处理用户/商家在端上发起的申诉申请，用户诉求主要以退款为主，商家诉求主要以餐损补贴为主。不同于智能客服实时处理和交互的方式，该场景处理的主要是迟滞工单，客服在该场景中的角色多以裁判员、协调员为主。

Agent 搭建模式：在该场景下我们搭建了 Agent + 人机协同的模式，Agent 在人工坐席辅助下，独立完成问题厘清 → 方案选择 → 沟通协商 → 判责 → 退款/驳回退款 → 触达通知 → 工单结案等全流程。

我的角色：统筹协调从方案设计到多团队拉齐，再到落地验证，贯穿全生命周期。一部分和工程共同设计一套人机协同的架构和规则协同，另一部分是人类专家（协同团队各角色）和 AI Agent 之间的翻译或转换。

实践实例

🤖 Agent 独立闭环 — "数字坐席"

用户因口味不佳申请退款，商家驳回，用户发起申诉。Agent 接到工单后，调用 skill 查找处置方案，外呼联系用户引导申请放心吃，用户同意，工单暂停等待放心吃结果。4小时后查询到已理赔，根据流程驳回退款，工单结案。

全程无需人工介入

👁️ 人帮 AI "看"

用户因餐品包装破损申诉。售后场景的关键是对举证凭证的核实：订单信息一致性、撒漏事实、是否影响食用、包装是否合规。人工坐席对图片关键信息的识别和确认，帮助 Agent 完成了判责流程的闭环。

人工补充感知能力

🔍 人帮 AI "找""看""沟通"

用户因未收到餐申诉。客商骑平台四方交替参与的时间线推理是 AI 强项。人工坐席帮助 Agent 在各渠道找送达照片、判断照片合规性、联系骑手索要凭证。

多渠道协作

二决策权在悄悄"让渡"

最初的设计，我们希望人工坐席在辅助的时候，看完图片就直接点选到责任方；但在和 BPI 沟通设计的时候，我们把人工坐席的点选项换成了图片中的关键信息核实项。

（人是决策主体）		 人给 AI 补信息，AI 做最终判断
（AI 是决策主体）
信息转换 → 决策让渡 是否是有效图片
责任方		 结构化转换：
• 订单信息和图片一致、撒漏属实
• 包装合规但撒漏 → 骑手责任
• 包装不合规但撒漏 → 商家责任

💡 人从"裁判"变成"AI 的眼睛和耳朵"，从"决策者"变成"感知执行者"。
本质上：人成为了 AI 的外挂传感器，AI 成为了决策的主导者。

三Agent 设计的反思：光会判不够，还要会协作、进化

决策权的让渡并不代表人的价值让渡。

人让出了什么：多方参与的复杂情况叠加时间线的推理——人在处理时耗费大量时间且容易出错；基于固定规则的重复性决策——每一个新的工单都可能是一次重复操作。但这些恰恰是 AI 最擅长的。

人拿回了什么：现阶段 AI 做不了的高价值能力：

能力表现
感知判断基于经验对图片关键信息的识别、核实、辨真伪
谈判协商柔和沟通、听语气、灵活应变，权衡用户、商家、骑手各方利益的谈判技巧

反过来看 Agent 的设计，人机协同不只是流程设计问题，它对 Agent 的能力架构提出了新要求——Agent 不仅需要"决策智能"（规则匹配、信息推理），还需要"协作智能"（跟人配合的能力）。

Agent 需要的五种协作能力

  • 🎯
    自我边界感知

    知道自己"不行"——识别哪些 case 信息不够、规则覆盖不了、置信度不高

    ⚠ 缺失后果：要么硬判出错，要么全丢给人（等于没用）
  • ❓
    精准提问

    会向人"下任务"——不是把整个 case 丢过去，而是告诉人"我已知什么、还缺什么、你只需做什么"

    ⚠ 缺失后果：人补位效率低，等于回到纯人工
  • 💬
    理解人话

    能消化非结构化回传——人工回传信息要能转换为 Agent 可推理的信号

    ⚠ 缺失后果：人补了位但 Agent 用不上，链路断裂
  • 🔄
    闭环判断

    知道什么时候"该收"——信息够了继续判，不够再问一轮；信息不够不要陷入重复纠结，交接给人处理

    ⚠ 缺失后果：无限循环或过早闭环，结果不可控
  • 🧬
    自进化

    人补位的信息（图片识别信息、质检纠正记录、外呼沟通记录）是大量高质量的标注数据，反哺到 Agent 是优质的自迭代闭环

    ⚠ 缺失后果：仍旧依赖人工，人的价值陷入另一种重复

边界设计：AI 和人的优势互相补齐，协同作业

🏛️ 决策权边界

AI：拥有常规判责的"可替代性"决策权，高效处理海量标准化 Case。

人：保留对复杂 Case 的"一票裁决权"和最终兜底责任。

🤝 交互协议边界

AI 的"求助权"：主动发起边界跨越，清晰结构化表达困惑，向人求助。

人的"主动纠正权"：运营可在任何环节打断 AI 流程，强行接管。

📡 信息转化边界

人 → AI：结构化的增强信号，辅助 AI 继续执行。

AI → 人：自然语言的"决策理由"和"不确定性来源"，可追溯可查证。

⚖️ 责任归属边界

模型责任区：因模型自身能力不足导致的误判。

操作责任区：因人输入错误"增强信号"导致的误判。

协议设计责任区：因协同规则缺陷导致的误判。

💰 成本边界

AI 处理高价值、高难度信息的投入成本和人工处理的成本权衡（AI 作为成本纳入计算；平峰高峰期的灵活调控）。

ℹ️ 启示：当我们设计人机协同 Agent 时，不能只追"判得准"，还要追"协作得顺"。Agent 足够智能、灵活，AI 和人工是互相补位的关系，协作智能决定了人机系统的整体效率上限。

四生产关系的连锁反应

决策主体从人变成 AI，人的价值从"执行规则"迁移到"驾驭系统"——不是贬值，是升维，整条生产链上几乎所有角色都在变：

角色以前（人做判断）现在 / 未来（AI 决策）
BPI设计、优化人的服务流程和规则设计、优化 AI 能力边界 + 人工介入触发点
一线客服经验判断、独立处理高效采集 AI 所需信息，配合闭环
培训教业务规则和判责逻辑教如何配合 AI、识别 AI 错误
质检抽检人的判断对不对评估"AI + 人 + 流程"系统整体效果
人机协同策略 owner写人能理解的政策，靠培训落地写机器能执行的规则，改配置即生效
产研做工具给人用做 Agent 能力 + 人机交互协议，协作智能
TL / 管理者管人均单量、准确率管系统闭环率，排班基于 AI 溢出量

∑总结

随着技术的发展，人机协同关系中决策权的让渡已经在悄悄发生。我们要做的不是阻止它，或是感到紧迫，而是主动设计——让人从重复劳动中释放，转向 AI 做不了的高价值环节：感知、共情、协商、质疑、纠偏、进化。

不是"人被取代"，而是"人和 AI 各归其位"。依赖于各类角色的重塑和组织的重构。
