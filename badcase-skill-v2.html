背景与目标

AIDA 是外商客服 AI 系统，底层采用 ReAct Agent 架构，执行路径为：

T
Thought
推理思考
→
A
Action
调用工具/Skill
→
O
Observation
获取结果
→
R
Response
生成回答

系统高度依赖 Skill（业务知识库） 回答商家问题。V1 实现了 LLM 自动分析 Badcase 并产出修改建议，但缺乏闭环——没有离线评测、没有迭代优化。

💡 V2 核心目标：引入离线评测 + 自动迭代机制，实现线下「改 → 测 → 改」闭环收敛，摆脱对人工校验的依赖。
🔄

🚀

关键创新点

  • 双轨评测体系 — A 类验修复 + B 类防误伤，确保优化不引入新问题
  • 自动迭代闭环 — LLM 驱动的「分析 → 修改 → 评测 → 再修改」循环，无需人工介入
  • 多维根因分析 — 知识遵循 / 上下文一致性 / 工具调用 / 沟通话术四维度精准定位问题
  • 用户模拟器 — 真实还原对话场景，评测结果更贴近线上表现
  • 守护集机制 — 引入线上真实人人对话作为 B 类守护集，防止 Skill 修改产生副作用
