子链路 7 层结构：

1
入口与准入
工单存在性校验 → 业务类型准入判断 → 不满足条件即返回"不准入"
2
信号加载 7 个 API 并发
内含信号加载子模型（appId: 7151f848-...）。并发请求工单详情、用户信息、订单状态等 7 路信号数据
3
AB 实验分流
根据实验配置将请求分配到实验组 / 对照组，对照组直接返回"不准入"
4
轮次管理 对话历史 Agent
内含对话历史 Agent（appId: 6c145a5e-...）。双路径架构：有历史 → 解析上文；无历史 → 初始化对话状态。轮次熔断：超过阈值则触发转人工
5
首轮 / 非首轮分流
首轮：返回固定欢迎话术，结束处理；非首轮：进入后续模型推理流程
6
用户意图识别 + 未结案处理
判断用户是否在咨询工单问题。未结案催促场景 → 发送催促卡片（空回复）。含对照组分流逻辑
7
次轮模型推理 LLM
模型：deepseek-v4-flash-tencent · 总结工单摘要（附录A Prompt）→ 模型推理生成回复 → 返回"准入处理"结果
